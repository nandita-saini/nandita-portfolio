IoT Data Logger Platform
High-Performance Data Ingestion · Laravel + MySQL
The Problem

The client operated 300+ IoT sensors across industrial sites sending data every few seconds. The existing system suffered from query timeouts, data loss under concurrent load, and dashboard lag of 15–30 seconds. The backend could not scale — adding new devices caused cascading slowdowns across the entire platform.

My Role & Contribution
  • Re-architected the data ingestion pipeline from individual row inserts to batch inserts — reducing DB write load by 80%
  • Profiled all slow queries using MySQL EXPLAIN and added composite indexes on (device_id, recorded_at) — the most common filter pattern
  • Built a paginated REST API for device data retrieval, replacing full-table scans with keyset pagination
  • Designed a partitioned table strategy by device group and date range for long-term data archival
  • Implemented a dashboard summary endpoint with pre-aggregated stats cached per device (5-min TTL)
Key Technical Decisions
  • Batch inserts over individual inserts: grouped incoming device payloads into 50-record chunks, reducing round trips from 300/s to 6/s
  • Composite indexing on high-cardinality filter columns eliminated full-table scans on the 10M+ row data log table
  • Used keyset pagination instead of OFFSET to keep retrieval O(log n) regardless of page depth
  • Separated hot (recent 24h) and cold (historical) data into different query paths to avoid index bloat

Role-Based Employee Performance Tracker · CodeIgniter + MySQL
The Problem

The company tracked employee performance entirely through spreadsheets — manually updated by managers, prone to errors, with no visibility for employees into their own scores. Generating a monthly performance report took 4–5 hours of manual work. There was no standardized formula, so scores differed between managers, creating fairness complaints.

My Role & Contribution

  • Plugged into Perfex's existing auth and role system rather than building a separate one — reduced scope by 40%

AI Call Processing System
Call Summarization & Transcript Analysis · PHP + AI APIs + n8n
The Problem

A support team was manually listening to and summarizing 200+ customer calls per day — an 8+ hour/day task split across 3 staff members. Summaries were inconsistent in quality, keywords were missed, and there was no searchable record of call outcomes. The team needed a way to automate call analysis without replacing the human review step entirely.

My Role & Contribution
  • Designed the end-to-end pipeline: Telephony system → Webhook → PHP ingestion layer → AI API → n8n routing → Database storage
  • Built the PHP webhook receiver that validates incoming call payloads, queues them for processing, and handles retry logic on AI API failures
  • Integrated with an AI transcription + summarization API — extracting call intent, sentiment, key action items, and outcome tags
  • Built n8n automation workflows to route processed call data: escalations to Slack, routine calls to archive, complaints to CRM ticket creation
  • Created a searchable admin dashboard showing call summaries, filterable by date, agent, sentiment, and outcome tag

  • Retry with exponential backoff on AI API calls — handles transient failures without losing call records

  • Stored raw transcripts alongside AI summaries — allowing re-processing with improved models without re-ingesting call audio

AI Security Optimization Project
Application Security Hardening · PHP + Security Tooling
The Problem

A PHP web application was flagged with active security exploits in production — SSRF vulnerabilities being used to probe internal services, malware injected into uploaded file handlers, API keys hardcoded in source files (committed to a shared repository), and multiple XSS vectors in user-facing input fields. The client needed immediate remediation with zero downtime.

My Role & Contribution
  • Conducted a full codebase security audit — manually reviewed every user-input point, HTTP client call, and file-handling routine
  • Patched SSRF vulnerabilities by implementing a strict URL allowlist for all outbound HTTP requests — blocking internal IP ranges and private subnets
  • Removed and rotated all hardcoded API keys; migrated secrets to environment variables with documented key rotation policy
  • Eliminated malware payloads injected into file upload handlers; added server-side MIME validation, extension whitelisting, and virus scan integration
  • Applied output encoding (htmlspecialchars) consistently across all user-data render points to close XSS vectors
  • Added Content Security Policy (CSP) headers and validated all form inputs server-side against strict type and length rules
Key Technical Decisions
  • Used a URL allowlist (not blocklist) for SSRF prevention — blocklists can be bypassed via DNS rebinding; allowlists cannot
  • Secrets moved to .env with a .gitignore guard and a pre-commit hook that fails if API key patterns are detected in staged files
  • Server-side MIME validation over client-side only — client content-type headers are trivially spoofed
  • CSP headers set to report-only first to collect violation data, then enforced after confirming no false positives
